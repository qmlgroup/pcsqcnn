# `qcnn` Concepts and Architecture

This document collects the repository-wide concepts, architecture, model
families, data/state conventions, and article-alignment notes.

For environment setup, build commands, and workflow instructions, see
`BUILD.md`.
For the detailed description of generated artifacts, payloads, and rendered
figure outputs, see `ARTIFACTS.md`.

## Runtime Overview

Training uses Torch as the primary engine.

- Torch provides direct GPU acceleration for dense and batched tensor algebra.
- Complex-valued autograd is explicit and easy to inspect.
- Block-diagonal and grouped linear operators can be implemented directly with tensor reshapes and batched linear algebra.
- Memory layout, dtype control, and performance tradeoffs stay under our control.

PennyLane is included only as an optional reference backend.

- It is inference-only.
- It is written for readability and article auditability, not speed.
- It loads Torch PCS-QCNN weights from this repository and reproduces Torch full readout probabilities and final logits.
- It keeps the visible article layer order `QFT -> multiplexer -> QFT†`.
- It does not support training, finite-shot sampling, reduced-readout mode, or the no-QFT ablation.

## Implemented Pipeline Capabilities

The pipeline currently provides the following capabilities as stable produced
behavior, not as planned work.

- Fixed translated-MNIST training for `ClassicalCNN`, `ClassicalMLP`, `PCSQCNN`, and `PCSQCNNNoQFT`.
- A no-QFT PCS-QCNN ablation implemented by `qcnn.hybrid.PCSQCNNNoQFT`, where each quantum layer uses a fixed random unitary `R` and its conjugate transpose `R*` instead of `QFT` and `IQFT`.
- A translated-MNIST PCS-QCNN architecture sweep over feature-qubit count and quantum-layer count.
- A full-MNIST PCS-QCNN image-size sweep over synchronized `(scaled_image_size, image_size)` pairs, including one canonical reference subrun with dense snapshots.
- A translated-digit brightness-range sweep over encoder upper-bound values `b` in `brightness_range=(0, b*pi)`.
- One-shot finite-shot reevaluation of saved checkpoints over multiple shot budgets, producing snapshot-level accuracy payloads.
- Repeated finite-shot reevaluation of saved checkpoints over fixed batch partitions, producing batch-level loss-sampling payloads.
- Layer-gradient diagnostic extraction on saved checkpoints.
- Sampled-readout entropy extraction on saved checkpoints.
- Local readout-landscape evaluation on an epoch-local PCA basis derived from test-averaged shot-noise covariance.
- Error-structure extraction from one saved final checkpoint, including both confusion matrix and misclassified-example gallery inputs.
- PDF rendering for every autogenerated figure panel in the paper and supplement.

## API Contract Conventions

The library treats documentation as part of the engineering contract.

- Module docstrings define the architectural role of each subsystem.
- Class and helper docstrings spell out tensor shapes, dtype expectations, and returned objects.
- Transform helpers include the formulas that define their contract, not only prose descriptions.
- Internal Fourier/junction/pooling helpers are documented on purpose: they are part of the PCS-QCNN architecture, not incidental implementation details.
- Performance notes are included only where they matter operationally, for example cached Pauli bases, batched matrix exponentials, FFT pairs, and reshape-only pooling.

## Canonical Tensor Contract

The authoritative wavefunction layout is:

```text
[B, X, CX, Y, CY, F]
```

Axis semantics:

- `B`: batch axis
- `X`: active `x`-register basis states
- `CX`: explicit `x`-condition register basis states
- `Y`: active `y`-register basis states
- `CY`: explicit `y`-condition register basis states
- `F`: feature-register basis states

Bit order is MSB-left inside every logical register. The encoder starts from the
special case `[B, X, 1, Y, 1, F]`, and deferred-measurement pooling grows the
explicit condition axes from there while preserving the outer axis order above.
Each newly pooled qubit becomes the most-significant bit of its condition
register, so the most recently measured outcomes appear first.

The grayscale/color qubit is the least-significant feature bit. Auxiliary
feature qubits, when present, are initialized to `|0...0>` so only feature
indices `0` and `1` are populated by the encoder.

## Encoder Contract

`FrqiEncoder2D` implements the FRQI-like brightness map

```text
|phi_uv> = sin(p_uv)|0> + cos(p_uv)|1>
```

For normalized grayscale inputs `x_uv in [0, 1]`, the encoder first computes
`p_uv = a + (b - a) * x_uv` using its configurable `brightness_range = (a, b)`
and then applies the map above. The encoder accepts input tensors only in shape
`[B, X, Y]`.

Important: the encoder intentionally does **not** apply the article's global
`1 / sqrt(XY)` normalization factor from the FRQI-like state definition. This
is a deliberate v1 mismatch with the article. As a result, encoded sample norms
are `sqrt(XY)`, not `1`.

This mismatch is surfaced in three places:

- this document
- the `FrqiEncoder2D` docstring
- runtime warnings emitted when the encoder is instantiated

## Measurement Readout

`MarginalMeasurement2D` converts a canonical complex state in axis order
`[B, X, CX, Y, CY, F]` into a real readout tensor `[B, X, Y, F]`.

The layer:

- computes pointwise probabilities from `|psi|^2`
- sums over the deferred-measurement condition axes `CX` and `CY`
- divides by the known shape factor `X * CX * Y * CY`

This keeps the encoder contract unchanged while producing normalized marginal
readout probabilities at measurement time for encoder-style locally normalized
states.

The measurement contract is

```text
m[b, x, y, f] = (1 / (X * CX * Y * CY)) * sum_{cx, cy} |psi[b, x, cx, y, cy, f]|^2
```

`FiniteShotHistogram2D` can optionally sit on top of that exact readout. When
configured with a finite `shot_budget`, it samples a multinomial histogram and
returns normalized frequencies with the same `[B, X, Y, F]` shape. The layer is
disabled by default and also stays inactive while the surrounding model is in
training mode so the standard Torch training loop remains differentiable.

## PennyLane Reference Contract

`pennylane/pcs_qcnn_reference.py` is an optional readable reference backend,
not the main runtime.

- The supported parity target is Torch full readout `[B, X_active, Y_active, F]` and final logits.
- The reference loads Torch `PCSQCNN` constructor kwargs plus `state_dict()` weights.
- The reference always executes the explicit readable PCS layer stack with built-in `qml.QFT` and `qml.adjoint(qml.QFT)`.
- The reference keeps large mode-dependent feature blocks as high-level unitaries instead of decomposing them into elementary gates.
- The reference intentionally does not implement Torch-side runtime shortcuts such as the reduced Fourier junction.
- `reduce_readout_to_feature_distribution=True` remains a Torch-only mode and is outside the PennyLane/article contract.

The encoder mismatch remains explicit:

- Torch omits the article's global `1 / sqrt(XY)` normalization factor.
- The PennyLane reference uses normalized `StatePrep`.
- Required cross-backend agreement is at measured full readout and logits, not at intermediate amplitudes.

## Model Families

### `PCSQCNN`

`PCSQCNN` is a multilayer PCS-QCNN with a classical logits head.

Its current architecture is:

- `FrqiEncoder2D`
- repeated quantum layers
- `MarginalMeasurement2D`
- optional `FiniteShotHistogram2D`
- flatten to `[B, X_active * Y_active * F]`
- `nn.Linear(..., num_classes)` returning logits

Each quantum layer applies:

- `QFT_x/QFT_y` on the active index registers
- `ModeMultiplexer2D`, which applies a feature-register unitary block for each active `(x, y)` mode and selected condition branch
- `IQFT_x/IQFT_y`

The feature-unitary blocks are parameterized as

```text
H(theta) = sum_alpha theta_alpha P_alpha
U(theta) = exp(i * H(theta))
```

with the reduced multi-qubit Pauli basis on the feature register.

After every non-final layer, one active qubit from each index register is moved
into the corresponding condition register via deferred measurement. The first
multiplexer is unconditional. Every later multiplexer is conditioned on exactly
one bit per register: the most recently measured `x` bit and the most recently
measured `y` bit.

By default, `PCSQCNN(..., use_reduced_fourier_junction=True)` replaces every
adjacent `IQFT -> move_active_qubit_to_condition -> QFT` triple with the
analytically equivalent reduced Fourier junction helper. Setting the flag to
`False` keeps the explicit composition.

After the final `IQFT`, the model does not perform another pooling step.
Instead it measures the whole remaining state and uses `MarginalMeasurement2D`
to sum over all accumulated condition branches before the linear head.

`PCSQCNN(..., shot_budget=None)` keeps the existing exact-probability readout.
Setting a finite `shot_budget` enables multinomial sampling only in `.eval()`
mode; `.train()` still forwards exact readout probabilities into the classical
head.

Loss computation stays outside the model via `nn.CrossEntropyLoss()`.

### `PCSQCNNNoQFT`

`PCSQCNNNoQFT` is the no-QFT ablation used by the article pipeline.

Each quantum layer uses a fixed random unitary `R` and its conjugate transpose
`R*` instead of `QFT` and `IQFT`. It is not a learned QFT replacement: the
sampled spatial unitary is stored as a buffer and remains non-trainable.

### `ClassicalCNN` and `ClassicalMLP`

`ClassicalCNN` and `ClassicalMLP` provide the classical MNIST references while
matching the `PCSQCNN` public boundary:

- both accept `[B, image_size, image_size]`
- both return logits `[B, num_classes]`
- neither applies a final softmax

`ClassicalCNN` now uses a stronger translated MNIST architecture: four `3x3`
convolutions grouped into two `Conv -> ReLU -> Conv -> ReLU -> AvgPool` stages,
followed by `AdaptiveAvgPool2d(1)`, `Dropout(0.10)`, and a final biased linear
classifier. The article training script fixes `base_channels=16` explicitly in
its saved model specs so benchmark artifacts remain tied to this exact CNN.

`ClassicalMLP` now uses a stronger pure-dense baseline: flattening followed by
three `Linear -> GELU -> Dropout(0.10)` stages and a final linear classifier.
Its default hidden widths are resolved automatically from `image_size` so the
trainable-parameter budget stays comparable to the CNN baseline, and the
article training script saves those resolved widths explicitly in its model
specs. Both classical baselines now consume the normalized `[0, 1]` image
tensors produced by `prepare_mnist_splits` directly; they do not apply the
quantum-specific `brightness_range` mapping.

## Training and Evaluation Interfaces

### `ImageClassifierRunner`

`ImageClassifierRunner` is the recommended notebook-facing wrapper for the
article models and other grayscale image classifiers that consume `[B, X, Y]`
inputs and return logits.

The runner is DataLoader-first and provides:

- `run_forward_pass`, `run_backward_pass`, and `run_training_step` for batch-level work
- `evaluate_loader` for sample-weighted loss/metric evaluation on a loaded model
- `train_epoch` and `fit` for compact training loops with epoch-level history

Built-in `accuracy_from_logits` is included by default, and additional custom
scalar metrics can be registered on the runner without changing the wrapped
model or the DataLoader contract. When `loss_fn` is omitted, the runner uses
`nn.CrossEntropyLoss()` by default. After a successful `fit(...)`, the runner
also retains the latest train/test loaders and the returned `TrainingHistory`
so saved run metadata can be reconstructed without requiring the caller to
re-specify those settings.

### Serialization

Automated runs save two complementary files per seed:

- `checkpoint_*.pt` stores reconstruction-ready `model_spec + model_state`
- `result_seed*.pt` stores the lightweight experiment summary with resolved config, saved loss name, training history, collector states, and final train/test summaries

The primary reconstruction helpers are:

- `load_model_checkpoint(path)` to rebuild a model from a checkpoint bundle
- `load_auto_training_run(run_directory, seed)` to rebuild a model together with its saved result payload
- `evaluate_loaded_auto_training_run_on_saved_mnist_test(...)` to re-run the saved MNIST evaluation flow from `checkpoint_final + result_seed`

This format intentionally keeps weights out of `result_seed*.pt`. Optimizer
state is not stored for training resume.

### Visualization Helpers

The library also exposes two notebook-facing plotting helpers:

- `plot_convergence(history, metric_name="accuracy")` for the standard two-panel training curve figure
- `plot_error_analysis(runner, data_loader)` for a confusion matrix plus a gallery of misclassified images in the article style

These helpers import `matplotlib` lazily, so the base package keeps plotting as
an optional extra rather than a hard runtime dependency.

## Data and Preprocessing

### Shared Training Defaults

Unless a family overrides them explicitly, article-oriented training uses these
defaults from `qcnn.article_training.ArticleTrainingDefaults`.

- `optimizer_kind="adam"`
- `learning_rate=1e-2` for classical models
- `hybrid_learning_rate=3e-2` for hybrid PCS-QCNN models
- `weight_decay=0.0`
- `momentum=0.0`
- `num_workers=0`
- `pin_memory=False`
- `download=True`
- `set_to_none=True`
- `test_evaluation_interval_epochs=10`
- `snapshot_epochs=()`
- `device=None`, which resolves to CUDA when available and CPU otherwise
- `torch_matmul_precision="high"`
- `multiplexer_init_scale=0.05`
- `base_seed=0`
- `seed_count=1` at the base class level, overridden by the article run families below
- `use_timestamp_dir=False`

### Dataset Conventions

- When `samples_per_class` is an integer, the dataset is a balanced MNIST subset with that many training samples per class.
- When `samples_per_class=None`, the full standard MNIST train split is used.
- `scaled_image_size=None` resolves to `image_size` inside the saved dataset config.
- Test metrics are only recorded on the epochs where test evaluation runs. With the current default interval this means every 10 epochs and the final epoch.
- Train metrics are recorded every epoch.

### MNIST Preprocessing

The library provides deterministic MNIST preparation utilities.

The current benchmark uses two MNIST regimes:

- full MNIST: keep the standard train/test split and reuse the existing figure-specific preprocessing choices without semantic changes
- translated MNIST: use a balanced subset with `1000` training examples per class, resize digits from `28x28` to `16x16`, place them on a `32x32` canvas, sample seeded integer translations from `[-8, 8]`, and keep the standard MNIST test split membership unchanged

The library-level article-alignment warnings compare configurations against
the article's translated-benchmark sample-count convention while retaining the
generic centered-resize preprocessing defaults for the low-level data helper:

- `samples_per_class = 1000`
- `image_size = 16`
- `scaled_image_size = image_size`
- `max_offset = 0`
- bilinear resize onto a centered canvas
- grayscale normalization to `[0, 1]`
- standard MNIST test split membership kept unchanged

Passing `samples_per_class=None` selects the full standard MNIST training split
instead of a class-balanced subset regime while preserving the same generic
preprocessing pipeline.

The preparation pipeline:

1. Either select exactly `samples_per_class` random training samples per class using a seeded split, or keep the full train split when `samples_per_class=None`.
2. Keep the standard MNIST test split membership unchanged.
3. Resize every image to `scaled_image_size` with bilinear interpolation.
4. Place every resized image onto the final `image_size x image_size` canvas.
5. Optionally translate the digit by seeded integer offsets drawn independently per axis from `[-max_offset, max_offset]`.
6. Normalize grayscale values into `[0, 1]`.

If `image_size`, `samples_per_class`, or the optional translation pipeline
deviate from these helper-level alignment conventions, the data-preparation API
emits runtime warnings. The article-aligned encoder angle range remains
`brightness_range = (0, pi)` and is now configured on `FrqiEncoder2D` /
`PCSQCNN`.
Prepared `TensorImageDataset` instances also carry lightweight metadata about
their split and preprocessing settings so artifact serialization can recover the
relevant experiment context from the runner alone.

## Article Alignment and Intentional Mismatches

Matched behaviors:

- grayscale-only MNIST preprocessing
- bilinear interpolation followed by centered placement on a power-of-two canvas
- normalization of prepared image tensors into `[0, 1]`
- standard MNIST test split membership remains unchanged
- article-aligned warning baseline for the translated benchmark sample count: `1000` samples per class
- article-aligned encoder angle range: `(0, pi)`
- FRQI-like local brightness map `sin(p)|0> + cos(p)|1>`

Intentional mismatch:

- optional translated preprocessing can decenter digits relative to the article's centered resize-only pipeline
- the encoder omits the article's global `1 / sqrt(XY)` normalization factor
- the measurement layer restores normalized marginal readout by dividing the final probabilities by the known shape factor, without changing the encoder itself

How mismatches are surfaced:

- explicit notes in `README.md`, `INFO.md`, and docstrings
- runtime warnings from the library
- notebook markdown that calls out the mismatch before encoding any batches
