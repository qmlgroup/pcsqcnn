# Article Artifact Contract

## Overview

This file defines the contract for the autogenerated article artifact pipeline.

- This document covers generated training artifacts, derived reevaluation payloads, per-task caches, and rendered PDFs.
- The public precomputed artifact bundle is currently distributed through the
  [`artifacts-v1` GitHub release](https://github.com/qmlgroup/pcsqcnn/releases/tag/artifacts-v1).
- When using that release locally, unpack its archive into the repository-root
  `artifacts/` directory.
- Rendered PDFs are produced in `figs/`.

The intended reading model is: if a script, payload, or PDF name appears here, downstream code may rely on the stated path, field names, and semantics as a contract.

## Shared conventions

### Directory roots

- Training data root: `data/`.
- Generated training and reevaluation artifacts root: `artifacts/`.
- Generated figure root: `figs/`.
- Article-facing copied figure root: `figs/`.

### Automated run directory contract

Every training family below is produced through `run_auto_training_manifest_tasks(...)` and therefore uses the same run-directory structure.

- Each run family uses an explicit `output.directory_name`, so article runs are not timestamped.
- Each run directory contains one `manifest.json` file, one `result_seed{seed}.pt` file per saved seed, one `checkpoint_final_seed{seed}.pt` per seed, and zero or more `checkpoint_epoch{epoch}_seed{seed}.pt` snapshot files.
- `manifest.json` top-level fields are:
  - `output_directory`: absolute path of the run directory.
  - `device`: resolved execution device used for training.
  - `resolved_config`: plain-data form of the global automated-training configuration.
  - `seeds`: ordered saved seed list.
  - `runs`: one per-seed entry.
- Each `manifest.json` run entry contains:
  - `seed`
  - `result`
  - `checkpoint_final`
  - `snapshots`
  - `final_train_summary`
  - `final_test_summary`
- The `seeds` list is the authoritative seed ordering. All figure loaders consume training histories in that order.
- Single-seed reevaluation scripts default to the first seed in `manifest.json` when `--seed` is not passed.

### Per-seed result payload contract

Each `result_seed{seed}.pt` payload stores the lightweight training summary for one seed.

- Top-level fields are:
  - `resolved_config`
  - `seed`
  - `device`
  - `loss_name`
  - `parameter_stats_line`
  - `final_summaries`
  - `training_history`
  - `collector_states`
  - `files`
- `files.result` repeats the result filename.
- `files.checkpoint_final` names the final checkpoint file for the same seed.
- `files.snapshots` maps saved epoch numbers, serialized as strings, to checkpoint filenames.
- `training_history` is the canonical source for train and test metric traces used by the Figure 2, brightness-sweep, Figure 5a, and Figure 5b renderers.

### Figure aggregation conventions

- `ARTICLE_PANEL_FIGSIZE` is `(6, 4)` and is used by all autogenerated numeric panels except the classical architecture schematics.
- `summarize_accuracy_histories(...)` computes per-epoch mean and population standard deviation across saved seeds.
- Figure 2 panels use test bands equal to `mean ± std` across seeds.
- The brightness-sweep, Figure 5a, and Figure 5b renderers use `mean`, `min`, and `max` across seeds.
- The Figure S1 architecture schematics are the explicit exception to the fixed panel-size rule. They use content-sized layout generated from the current model definitions.

### Canonical reference run

The canonical full-MNIST reference run is the `16on16` member of the image-size sweep.

- Run directory: `artifacts/pcsqcnn_image_size_sweep/16on16`
- Dataset regime: full MNIST, `samples_per_class=None`
- Dataset geometry: `scaled_image_size=16`, `image_size=16`, `max_offset=0`
- PCS-QCNN architecture: `feature_qubits=3`, `quantum_layers=1`
- Encoder brightness range: `(0.0, pi)`
- Readout mode: full readout, not reduced-to-feature mode
- Training mode: exact-probability training and exact-probability checkpoints
- Dense snapshot schedule: `(10, 100, 200, ..., 2000)`

This run is the default upstream source for:

- `readout_entropy.pt`
- `finite_shot_accuracy.pt`
- `finite_shot_loss_sampling.pt`
- `readout_landscape.pt`
- `error_structure_seed{seed}.pt` when the default `16on16` size pair is used

    - `classical_cnn_spc1000`
    - `pcsqcnn_no_qft_spc1000`
    - PCS-QCNN reference for Fig. 2(b): `pcsqcnn_architecture_sweep_full_readout/fq2_ql3`
  - Directory naming rule: `{base_name}_spc{samples_per_class}`
- Upstream inputs and source data:
  - MNIST train/test data reconstructed under `data/`
  - Balanced translated-MNIST preprocessing applied with the saved seed for each run
- Exact current defaults from code:
  - `samples_per_class=1000`
  - `image_size=32`
  - `scaled_image_size=16`
  - `max_offset=8`
  - `train_batch_size=256`
  - `test_batch_size=1600`
  - `num_epochs=2000`
  - `seed_count=3`
  - `test_evaluation_interval_epochs=10`
  - Quantum baseline settings:
    - `feature_qubits=2`
    - `quantum_layers=1`
    - `brightness_range=(0.0, pi)`
    - `reduce_readout_to_feature_distribution=False`
  - Classical CNN model spec:
    - `ClassicalCNN(image_size=32, num_classes=10, base_channels=16, dropout=0.10)`
  - Classical MLP model spec:
    - `ClassicalMLP(image_size=32, num_classes=10, hidden_widths=resolve_classical_mlp_hidden_widths(32, num_classes=10), dropout=0.10)`
  - No-QFT model spec:
    - `PCSQCNNNoQFT(image_size=32, num_classes=10, feature_qubits=2, quantum_layers=1, brightness_range=(0.0, pi), shot_budget=None, reduce_readout_to_feature_distribution=False, multiplexer_init_scale=0.05)`
- Stored contract:
  - Each run directory follows the shared automated run-directory layout.
  - `training_history.train_metrics["accuracy"]` is the source for train curves.
  - `training_history.test_metrics["accuracy"]` is the source for test curves.
  - `parameter_stats_line` is stored in every seed payload and must be identical across seeds within one run directory.
- Computation contract:
  - All four model families use the same translated-MNIST dataset regime and the same seed schedule.
  - `PCSQCNNNoQFT` is not a learned QFT replacement. Each layer samples one fixed random spatial unitary `R`, stores it as a buffer, applies `R -> multiplexer -> R*`, and leaves `R` itself non-trainable.
- Downstream consumers:
  - `run/plot_translated_mnist_baselines.py`
  - `run/plot_classical_baseline_architectures.py` uses the same Figure 2 model specs, but does not read the saved training artifacts
  - Rendered PDFs:
    - `translated_mnist_classical_baselines.pdf`
    - `translated_mnist_quantum_ablation.pdf`

## Derived evaluation payloads

### `gradient_norms.pt`

- Purpose: Stores the depth-scaling initialization-time RMS quantum-gradient diagnostic used for Figure S2a.
- Producer script: `run/evaluate_gradient_norms.py`
- On-disk location and filename contract:
  - Assembled payload path: `artifacts/depth_scaling_gradient_norms/gradient_norms.pt`
  - Per-task cache root: `artifacts/depth_scaling_gradient_norms/`
  - Per-task directory: `depth{depth:02d}_pseed{param_seed:02d}/`
  - Per-task files: `manifest.json` and `result.pt`
- Upstream inputs and source data:
  - Freshly instantiated random-initialization PCS-QCNN models
  - Deterministically prepared balanced-subset MNIST test samples at the depth-dependent image sizes
- Exact current defaults from code:
  - `depths=(1, 2, 3, 4, 5, 6, 7, 8)`
  - `param_seeds=(0, 1, ..., 11)`
  - `data_seed=0`
  - `post_pooling_index_qubits=1`
  - `feature_qubits=3`
  - `num_test_samples=256`
  - `class_balanced_subset=True`
  - `task_type="depth_scaling_gradient_norms"`
  - `format_version=2`
  - Local safety guard:
    - `LOCAL_SAFE_MAX_IMAGE_SIZE=256`
    - `LOCAL_SAFE_MAX_PARAM_SEED_COUNT=12`
    - `LOCAL_SAFE_MAX_NUM_TEST_SAMPLES=256`
    - `LOCAL_SAFE_MAX_TOTAL_WORK=3072`
- Stored contract:
  - Top-level payload fields:
    - `depths`
    - `param_seeds`
    - `data_seed`
    - `post_pooling_index_qubits`
    - `feature_qubits`
    - `num_test_samples`
    - `class_balanced_subset`
    - `evaluations`
  - Each `evaluations` entry contains:
    - `param_seed`
    - `depth`
    - `image_size`
    - `layer_keys`
    - `layer_labels`
    - `full_quantum_gradient_rms`: scalar tensor for all quantum parameters
    - `last_quantum_layer_gradient_rms`: scalar tensor for the last quantum layer only
  - Each task-cache manifest contains:
    - `task_type`
    - `format_version`
    - `saved_at_utc`
    - `param_seed`
    - `depth`
    - `image_size`
    - `scaled_image_size`
    - `max_offset`
    - `post_pooling_index_qubits`
    - `feature_qubits`
    - `num_classes`
    - `data_seed`
    - `num_test_samples`
    - `class_balanced_subset`
    - `result_filename`
- Computation contract:
  - For each depth/parameter-seed pair, the evaluator instantiates a fresh PCS-QCNN in the depth-scaling regime with `image_size = 2^(post_pooling_index_qubits + depth - 1)`.
  - The script selects one deterministic class-balanced test subset using `data_seed`, preprocesses it at the depth-dependent image size, and averages squared per-sample gradient norms over that fixed subset.
  - Per-sample gradients are evaluated with `torch.func.functional_call`, `torch.func.grad`, and `torch.func.vmap`, then reduced microbatch-by-microbatch to avoid storing the full Jacobian.
  - The adaptive microbatch fallback halves the current batch size after CUDA OOM until it reaches `1`; if that still fails, the script raises instead of writing a partial task result.
- Downstream consumers:
  - `run/plot_gradient_norms.py`
  - Rendered PDF:
    - `gradient_norms.pdf`

### `readout_entropy.pt`

- Purpose: Stores per-sample Shannon entropy values for sampled full-readout histograms over multiple shot budgets and checkpoints.
- Producer script: `run/evaluate_readout_entropy.py`
- On-disk location and filename contract:
  - Assembled payload path: `artifacts/pcsqcnn_image_size_sweep/16on16/readout_entropy.pt`
  - Per-task cache root: `artifacts/pcsqcnn_image_size_sweep/16on16/readout_entropy/`
  - Per-task directory: `epoch{epoch:03d}_shots{shot_budget}/`
  - Per-task files: `manifest.json` and `result.pt`
- Upstream inputs and source data:
  - The canonical `16on16` training run
  - Snapshot checkpoints at the requested epochs
  - The reconstructed saved MNIST test split
- Exact current defaults from code:
  - `epochs=(100, 200, ..., 2000)`
  - `shot_budgets=(128, 256, 512, 1024, 2048)`
  - `batch_size=128`
  - `task_type="readout_entropy"`
  - `format_version=1`
- Stored contract:
  - Top-level payload fields:
    - `seed`
    - `epochs`
    - `shot_budgets`
    - `evaluations`
  - Each `evaluations` entry contains:
    - `epoch`
    - `shot_budget`
    - `entropy`: 1D tensor with one Shannon entropy value per test sample
  - Each task-cache manifest contains:
    - `task_type`
    - `format_version`
    - `saved_at_utc`
    - `seed`
    - `epoch`
    - `shot_budget`
    - `batch_size`
    - `result_filename`
- Computation contract:
  - For each test sample, the evaluator builds one sampled full-readout histogram at the requested shot budget and computes Shannon entropy in nats.
  - The assembled payload stores the raw per-sample entropies, not only aggregated moments.
- Downstream consumers:
  - `run/plot_readout_entropy.py`
  - Rendered PDF:

    - `total_samples`
    - `result_filename`
- Computation contract:
  - For each requested epoch, the evaluator computes the test-averaged multinomial covariance `(diag(mean_p) - mean_outer) / sigma_shot_budget`.
  - The two largest positive eigenmodes define the local basis.
  - `direction_u` and `direction_v` are sign-canonicalized so the largest-magnitude component is positive.
  - The evaluator never renormalizes invalid perturbed distributions back onto the simplex. Negative-probability or overflowed samples are dropped from the cell average and counted through `valid_count`.
- Downstream consumers:
  - `run/plot_readout_landscape.py`
  - Rendered PDFs:
    - `readout_landscape_epoch10.pdf`
    - `readout_landscape_epoch800.pdf`
